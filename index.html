Introduction

The purpose of this page is to introduce those who are new to HTML, CSS and JS. The goal of this build is not to necessarily make the prettiest site there is out there but to learn some general concepts and leave a sort of "playground" for those learning to update and change their projects. The first aspect of this lesson will not be focused on mobile viewing.

I aim to make a lesson that will hopefully solidify these general concepts and arm those going through this a toolset to help them learn more after this lesson. I believe that by working on a build, we are helping ourselves understand these languages.

Lastly, I hope for this to be enjoyable as we see our build grow and respond to the various changes we will end up making to this page.

Comments

03/02/2021

Within HTML, we use < !-- [input message here ] --> to denote our comments. Makes both single and multi-line comments.

Within CSS, /* [message] */ allows for both single and multi-line comments

For JS, // [message] for single line comments and /* [message] */ for multi-line comments.

*quick note: using CTRL + / (on PC) & CMD + / (on Mac) is a quick way to comment out a line or multiple lines of code.

What are some reason why we want to add comments?

  • Notify your future self as to what you were building.
  • Clearly mark areas of your project.
  • Helps other developers navigate your project.
  • We can "clear" code to help us test while building.

The Terminal

03/01/21

  • View Current Directory:
    ls on mac / dir on PC.
  • Change your Directory:
    cd [name of folder]
  • Step Back or Out of Current Directory:
    cd ..
  • Making a new Directory:
    mkdir [name of folder]
  • Opening VSCode:
    code . (while sitting inside chosen folder/directory)

*Folder and Direcotry are the same in this example.

Git Lesson

03/01/21

  1. Initialize a Project:
    git init *done only once when connecting your project to github.
  2. Check Status:
    git status *optional task
  3. Selecting a Folder/Directory:
    git add .
  4. Create a Commit Message:
    git commit -m "your message here"
  5. Establishing Repo Connection:
    git remote add origin [paste repo here]*
  6. Push your Directory/Folder:
    git push origin main *older repos may use "master" instead of "main". The most up to date is "main"

*This connection link should only be needed when first establishing the repo/directory communication. When updating your project, you will only be required to do processes 3, 4, and 6
